# REPRODUCIBILITY.md

This document describes how to reproduce every number, figure, and table reported in
the manuscript, on free-tier hardware (any modern laptop with 8 GB RAM; GPU optional).

## 1. Environment

```
Python 3.11.15
Packages (full list in ai_autoboost/code/requirements.txt):
  numpy>=1.26, scipy>=1.11, pandas>=2.0, matplotlib>=3.8
  networkx>=3.2, statsmodels>=0.14, scikit-learn>=1.3
  torch>=2.2 (CPU build is sufficient)
  osmnx>=1.9, geopandas>=0.14
  tigramite>=5.2, joblib>=1.3
  pyyaml>=6.0, tqdm>=4.66
```

Install via:
```
python -m pip install -r ai_autoboost/code/requirements.txt
python -m pip install torch --index-url https://download.pytorch.org/whl/cpu
```

OS tested: Windows 10 / 11 (development), Linux glibc 2.31+ (expected to work).
No GPU required; the GraphSAGE surrogate is ~5 K parameters and trains in 32 min on CPU.

## 2. Hardware tested

- Development: AMD Ryzen 7-class, RTX 3070 (8 GB, used only for verification), 16 GB RAM.
- All scripts run start-to-finish in < 1 GPU-hour-equivalent on CPU.

## 4. Seed protocol

Every script accepts `--seeds N` (or uses an internal seed list with documented base
seed `2026_05_15`). All randomness flows through `numpy.random.default_rng(seed)` from
the orchestrator down; PyTorch uses `torch.manual_seed(seed)`. **No global numpy or
torch RNG state is left to default initialization.**

To verify bit-level reproduction:
```
sha256sum ai_autoboost/outputs/round4/expanded_cohort_summary.csv
# Reference: <hash to be added on final submission>
```

## 5. Live network dependencies

- **OSM (Overpass / Nominatim)**: required for R3/R4 OSM end-to-end. If OSM is
  unreachable, the framework falls back to synthetic archetypes (per R2 baseline).
- **USGS** event API (Christchurch retrospective only): if unreachable, the framework
  falls back to published Mw / R parameters (citations in r4_christchurch.py).
- **No paid APIs**.

## 6. Data sources

- CMIP6 SSP scenarios — climate-driver anchor values reproducible from any standard CMIP6
  ensemble (NEX-GDDP-CMIP6 publicly available on AWS).
- OpenStreetMap (osmnx pulls) — public.
- SoilGrids 2.0 — public (used implicitly via Vs30 surrogate calibrated to USGS Global
  Vs30 grid mean values).
- GLDAS-2 — public via NASA GES DISC (used as conceptual anchor for soil-moisture proxy).
- USGS earthquake catalog — public.

## 8. Validation checks

After running everything:

```
ls ai_autoboost/outputs/round4/expanded_cohort_summary.csv
ls ai_autoboost/outputs/round4/expanded_cohort_regression.csv  # expects R² ≈ 0.50, Pearson p ≈ 0.001
ls ai_autoboost/outputs/round4/loao_R2_stability.csv           # LOCO R² mean ≈ 0.50
ls ai_autoboost/outputs/round4/robustness_summary.csv          # R² in [0.47, 0.75]
ls ai_autoboost/outputs/round3/mediator_regression_summary.csv  # archetype R² ≈ 0.79
```

If any of these fail, the manuscript's headline numbers cannot be reproduced; please
file an issue at the project GitHub.

## 9. Known sources of non-determinism

- Live OSM fetches: the OSM database is mutable; running today vs in 6 months may yield
  slightly different node / edge counts. Our framework is robust to ±10 % node-count
  variation (verified in R4 graph-rewire perturbation up to 20 % with R² preserved).
- USGS API (Christchurch only): live API may re-catalog; the script falls back to
  published Mw / R if API returns implausible values.
